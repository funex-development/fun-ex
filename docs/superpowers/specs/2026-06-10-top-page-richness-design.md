# トップページ「ヒーロー以降」リッチ化 設計書

日付: 2026-06-10
対象: `public/index.html`（トップページ）のヒーローセクションより下
方針: 案A（モーション強化）＋案C（セクションの見た目刷新）の組み合わせ。ユーザー承認済み。

## 背景・目的

トップページはヒーロー（パーティクルアニメ＋グラデーションテキスト）だけがリッチで、
それ以降の About／事業内容／ニュース／お問い合わせは「見出し＋テキスト＋ボタン」中心で
シンプルすぎる印象を与えている。構成や文章は変えずに、視覚的なリズムと動きを加えて
ページ全体の質感をヒーローに揃える。

## スコープ

### 変更するファイル（3ファイルのみ）

| ファイル | 変更内容 |
|---|---|
| `public/index.html` | 演出用クラスの付与のみ（文章・リンク・構成は不変） |
| `public/style.css` | セクション背景・区切り・カード化・モーション用CSSの追加 |
| `public/script.js` | IntersectionObserver によるスクロールリビール処理の追加 |

### スコープ外

- 他ページ（about.html / services.html / news.html / privacy.html / contact）
- ヒーローセクション・ナビゲーション・フッター（現状維持）
- 文章・リンク先・セクション構成・Swiperカルーセルの仕組み
- コンテンツ追加（案B：実績・数字・お客様の声）は将来の別件

## デザイン仕様（案Cぶん）

ページの背景リズム: 白 → （斜め区切り） → ライトグレー → （斜め区切り） → 白 → グラデ帯 → フッター

### 1. About（私たちについて）
- 白背景のまま、背景にブランド3色（#ff6b6b / #4ecdc4 / #ffd93d）の淡い radial-gradient ブロブを配置
- 不透明度はおおむね 15〜20%。本文の可読性を損なわないこと
- `overflow: hidden` で section 外へのはみ出しを防ぐ

### 2. 事業内容（Services）
- セクション背景をライトグレー（#f1f5f9 系）の帯に変更
- 上下の境界は CSS による斜めの区切り（clip-path または linear-gradient）。追加マークアップは最小限
- ダークなサービスカード（#1e293b）がグレー帯の上で映えることを確認する

### 3. 最新ニュース（News）
- 白背景。ニュース項目を罫線区切りリストからカード型に変更
  - 枠線 + 角丸 + 左端にアクセントカラーのバー
  - 既存の `.news-item` マークアップを活かし、CSSでカード化する
- news.html（一覧ページ）のニュースは対象外。トップページのみ

### 4. お問い合わせ（Contact）
- フルワイドのグラデーション帯: `linear-gradient(135deg, #ff6b6b, #ff8e53)`（既存ボタンと同系）
- 見出し・本文は白文字、ボタンは白背景＋コーラル文字に反転
- 大きめの太字テキストでコントラスト比 3:1 以上（WCAG 大文字基準）を満たすことを確認する
- 装飾として半透明の白い円を背景に配置（さりげなく）

## モーション仕様（案Aぶん）

### スクロールリビール
- IntersectionObserver で要素が viewport に入ったら `.is-visible` を付与し、
  opacity 0→1 ＋ translateY(20px→0) でフェードイン
- 同一セクション内の兄弟要素には 0.1s 程度のディレイ差（stagger）
- 一度表示したら再アニメしない（unobserve）
- 対象: 各セクションの section-header、サービスカルーセル全体、ニュースカード、CTAブロック

### ホバー演出
- サービスカード: 浮き上がり（translateY）＋アクセント色ボーダーの点灯
- ニュースカード: 左アクセントバーが太く/伸びる＋わずかな浮き上がり
- ボタン: わずかな拡大（scale 1.03程度）＋影の強調

### 背景アニメ
- About のブロブは 10s 以上のゆっくりした float アニメ（transform のみ使用、repaint負荷を避ける）

## アクセシビリティ・SEO 要件

1. **JS無効でも全コンテンツが見える**: リビール対象を CSS で初期非表示にしない。
   JS 起動時に `<html>` 等へ `js-reveal` クラスを付け、そのクラス配下でのみ初期非表示＋アニメを適用する
2. **prefers-reduced-motion: reduce** ではリビール・ブロブ・ホバー浮き上がりの transform アニメを無効化
3. 文章・見出し階層・リンク・構造化データは一切変更しない（SEOへの影響なし）
4. アニメは transform / opacity のみ使用（layout を起こすプロパティは使わない）

## 実装上の注意

- `style.css` への追記はファイル末尾にセクションコメント付きでまとめる（既存ルールの編集は最小限）
- 既存の `.news-item` ホバー、`.section-desc` のインラインスタイルとの競合に注意（CLAUDE.md の CSS競合の項を参照）
- `script.js` への追記は既存の DOMContentLoaded パターンに合わせる
- 検証はローカル dev サーバー（port 3000、使用中なら3001）＋ Chrome 実画面で行う

## 成功基準

- ヒーローから下にスクロールしたとき、セクションごとに背景・動きの変化があり「シンプルすぎる」印象が解消されている
- 文章・リンク・カルーセル挙動が変更前と同一
- prefers-reduced-motion 環境とJS無効環境で内容が問題なく読める
- モバイル（<768px）でも崩れない
